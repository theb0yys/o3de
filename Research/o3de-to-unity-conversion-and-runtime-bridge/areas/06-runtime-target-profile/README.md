# Runtime Target Profile

## Current evidence state

The following remain unknown for the user's exact lawful FoA installation and branch:

- game version and build identity;
- selected distribution branch;
- Mono or IL2CPP runtime target;
- Unity player version;
- managed/native assembly layout;
- BepInEx distribution and exact version;
- Addressables, AssetBundle, `Resources`, or custom loading behavior;
- exact runtime APIs and log locations needed by a selected capability.

Community observations referenced only through the supplied report's opaque citations do not establish a
default IL2CPP target, a separate Mono branch, or a BepInEx version. No runtime target is selected.

## Future evidence boundary

Exact-profile capture is not part of provider discovery or Unity Editor interchange. It requires a separately
reviewed, bounded, read-only diagnostic plan after the FoA mapping/runtime-extension gate. That plan must name
the exact operator-provided installation scope, permitted observations, path and privacy treatment, output
schema, redaction, retention, legal basis, failure behavior, and evidence-review process.

It must not automatically discover installations, recursively scan disks, extract assets, copy assemblies,
inspect saves, or commit proprietary/private material. Lawful possession is necessary but does not itself grant
the SDK authority to inspect or persist data.

## Output state

Any future capture returns source observations or candidate evidence. It does not automatically select Mono or
IL2CPP, approve a loader, promote catalog facts, or authorize compilation, deployment, game launch, or runtime
mutation.
