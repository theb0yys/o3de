# Runtime and Acquisition

The common acquisition order is lawful local capture, pinned GitHub snapshots, then optional Merlin Workshop.
All routes emit the same candidate observation contract and surface conflicts; none can silently override exact
local-build evidence.

`GameInformationAcquisition` now fixes that order in a compiled provider-neutral contract. The exact pinned
Tainted Grail GitHub source may emit candidate observations for its bound commit. Local capture and Merlin are
contract-only until an exact lawful installation or Merlin source revision is separately qualified. Conflicts
are retained even when a higher-precedence provider exists; no route can auto-promote or grant runtime access.

Avalon Awakened's Kandra/Unity work is an outbound conversion and evidence source for later character authoring.
Mono and IL2CPP runtime functions remain separate packages behind the existing adapter/build/deploy/result gates.

`FoARuntimeAdapterRoutes` now preserves two distinct planning routes. Mono is bound to Tainted Framework 0.1.33
host live-load evidence; IL2CPP is bound to the later 0.1.36 package install/load and release-decision evidence.
Both target FoA 1.23.401 and Unity 6000.0.64f1 with their exact loader versions. These are compatibility/evidence
routes only: build, deployment, execution, game API calls, runtime mutation, and save access remain disabled.
